# Superhuman-cpa-OS Tax — Scaffold Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the full monorepo with all 5 service stubs running, shared types, complete DB schema, K8s manifests, and CI/CD — so any developer can clone, run `bun install`, and have everything booting within minutes.

**Architecture:** Domain-split Bun monorepo with 5 independently deployable services and one shared types package. Each service boots and responds to `GET /health`. The DB migration runner applies all schema migrations to SQLite in dev. No business logic yet — this is the foundation every subsequent plan builds on.

**Tech Stack:** TypeScript (strict), Bun workspaces, React + Vite + Tailwind, SQLite dev (bun:sqlite), Vitest, Kubernetes, GitHub Actions

- [ ] **Step 3: Write `services/db/migrations/001_initial.sql`**

All tables use SQLite-compatible types. JSON columns are stored as TEXT. Boolean columns use INTEGER (0/1). UUIDs are TEXT (generated in the application layer via `crypto.randomUUID()`).

```sql
-- Migration tracking table — always created first.
CREATE TABLE IF NOT EXISTS _migrations (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  filename   TEXT NOT NULL UNIQUE,
  applied_at TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS firms (
  id         TEXT PRIMARY KEY,
  name       TEXT NOT NULL,
  slug       TEXT NOT NULL UNIQUE,
  branding   TEXT,
  status     TEXT NOT NULL DEFAULT 'active'
               CHECK(status IN ('active', 'suspended')),
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS users (
  id                 TEXT PRIMARY KEY,
  firm_id            TEXT NOT NULL REFERENCES firms(id),
  email              TEXT NOT NULL,
  role               TEXT NOT NULL
                       CHECK(role IN ('taxpayer', 'pro', 'staff')),
  status             TEXT NOT NULL DEFAULT 'active'
                       CHECK(status IN ('active', 'pending_ptin_verification', 'suspended')),
  phone_number       TEXT,
  passkey_credential TEXT,
  password_hash      TEXT,
  created_at         TEXT NOT NULL DEFAULT (datetime('now')),
  UNIQUE(firm_id, email)
);

CREATE TABLE IF NOT EXISTS tax_returns (
  id             TEXT PRIMARY KEY,
  firm_id        TEXT NOT NULL REFERENCES firms(id),
  taxpayer_id    TEXT NOT NULL REFERENCES users(id),
  pro_id         TEXT REFERENCES users(id),
  tax_year       INTEGER NOT NULL,
  base_form      TEXT NOT NULL
                   CHECK(base_form IN ('1040', '1040-SR', '1040-NR', '1040-NR-EZ')),
  schedules      TEXT NOT NULL DEFAULT '[]',
  status         TEXT NOT NULL DEFAULT 'intake'
                   CHECK(status IN (
                     'intake', 'ai_prep', 'pending_pro_review', 'review', 'signed', 'filed'
                   )),
  payment_status TEXT NOT NULL DEFAULT 'unpaid'
                   CHECK(payment_status IN ('unpaid', 'paid', 'refunded')),
  data           TEXT NOT NULL DEFAULT '{}',
  filing_deadline TEXT,
  created_at     TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS intake_sessions (
  id         TEXT PRIMARY KEY,
  return_id  TEXT NOT NULL REFERENCES tax_returns(id),
  channel    TEXT NOT NULL
               CHECK(channel IN ('chat', 'voice', 'wizard', 'upload', 'phone')),
  messages   TEXT NOT NULL DEFAULT '[]',
  status     TEXT NOT NULL DEFAULT 'open'
               CHECK(status IN ('open', 'complete', 'abandoned')),
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS documents (
  id             TEXT PRIMARY KEY,
  return_id      TEXT NOT NULL REFERENCES tax_returns(id),
  doc_type       TEXT NOT NULL
                   CHECK(doc_type IN (
                     'w2', '1099-nec', '1099-misc', '1099-b', '1099-div',
                     '1099-int', '1099-r', 'k1', 'schedule-c-records', 'receipt', 'other'
                   )),
  storage_key    TEXT NOT NULL,
  extracted_data TEXT,
  verified       INTEGER NOT NULL DEFAULT 0,
  created_at     TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS pro_tasks (
  id           TEXT PRIMARY KEY,
  return_id    TEXT NOT NULL REFERENCES tax_returns(id),
  pro_id       TEXT REFERENCES users(id),
  gate_type    TEXT NOT NULL
                 CHECK(gate_type IN ('review', 'sign', 'advice', 'represent')),
  status       TEXT NOT NULL DEFAULT 'pending'
                 CHECK(status IN ('pending', 'in_review', 'approved', 'rejected')),
  notes        TEXT,
  completed_at TEXT,
  created_at   TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS orders (
  id                TEXT PRIMARY KEY,
  firm_id           TEXT NOT NULL REFERENCES firms(id),
  return_id         TEXT NOT NULL REFERENCES tax_returns(id),
  taxpayer_id       TEXT NOT NULL REFERENCES users(id),
  total_cents       INTEGER NOT NULL DEFAULT 0,
  status            TEXT NOT NULL DEFAULT 'pending'
                      CHECK(status IN ('pending', 'paid', 'partially_refunded', 'refunded')),
  payment_intent_id TEXT,
  created_at        TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS order_items (
  id           TEXT PRIMARY KEY,
  order_id     TEXT NOT NULL REFERENCES orders(id),
  product_type TEXT NOT NULL
                 CHECK(product_type IN (
                   'tax_preparation', 'audit_protection', 'refund_transfer', 'refund_advance'
                 )),
  amount_cents INTEGER NOT NULL,
  status       TEXT NOT NULL DEFAULT 'pending'
                 CHECK(status IN ('pending', 'paid', 'refunded', 'voided')),
  metadata     TEXT,
  created_at   TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS threads (
  id           TEXT PRIMARY KEY,
  firm_id      TEXT NOT NULL REFERENCES firms(id),
  return_id    TEXT REFERENCES tax_returns(id),
  channel      TEXT NOT NULL
                 CHECK(channel IN ('chat', 'async', 'email', 'phone', 'sms')),
  participants TEXT NOT NULL DEFAULT '[]',
  subject      TEXT,
  status       TEXT NOT NULL DEFAULT 'open'
                 CHECK(status IN ('open', 'resolved')),
  created_at   TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS thread_messages (
  id          TEXT PRIMARY KEY,
  thread_id   TEXT NOT NULL REFERENCES threads(id),
  sender_id   TEXT REFERENCES users(id),
  ai_assisted INTEGER NOT NULL DEFAULT 0,
  body        TEXT NOT NULL,
  read_at     TEXT,
  created_at  TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS notifications (
  id           TEXT PRIMARY KEY,
  firm_id      TEXT NOT NULL REFERENCES firms(id),
  recipient_id TEXT NOT NULL REFERENCES users(id),
  return_id    TEXT REFERENCES tax_returns(id),
  type         TEXT NOT NULL,
  channel      TEXT NOT NULL
                 CHECK(channel IN ('email', 'in_app', 'sms')),
  payload      TEXT NOT NULL DEFAULT '{}',
  sent_at      TEXT,
  read_at      TEXT,
  created_at   TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS call_sessions (
  id                TEXT PRIMARY KEY,
  intake_session_id TEXT REFERENCES intake_sessions(id),
  return_id         TEXT REFERENCES tax_returns(id),
  taxpayer_id       TEXT NOT NULL REFERENCES users(id),
  agent_id          TEXT NOT NULL REFERENCES users(id),
  twilio_sid        TEXT,
  status            TEXT NOT NULL DEFAULT 'scheduled'
                      CHECK(status IN ('scheduled', 'active', 'completed')),
  transcript        TEXT,
  extracted_data    TEXT,
  ai_assist_log     TEXT,
  duration_seconds  INTEGER,
  recording_url     TEXT,
  created_at        TEXT NOT NULL DEFAULT (datetime('now'))
);

import { Database } from 'bun:sqlite';
import { readdir, readFile } from 'fs/promises';
import { join } from 'path';

const MIGRATIONS_DIR = join(import.meta.dir, 'migrations');

export async function applyMigrations(db: Database): Promise<void> {
  // Ensure _migrations table exists before reading it
  db.exec(`
    CREATE TABLE IF NOT EXISTS _migrations (
      id         INTEGER PRIMARY KEY AUTOINCREMENT,
      filename   TEXT NOT NULL UNIQUE,
      applied_at TEXT NOT NULL DEFAULT (datetime('now'))
    )
  `);

  const applied = new Set(
    (db.query('SELECT filename FROM _migrations').all() as { filename: string }[]).map(
      (r) => r.filename,
    ),
  );

  const files = (await readdir(MIGRATIONS_DIR)).filter((f) => f.endsWith('.sql')).sort();

  for (const filename of files) {
    if (applied.has(filename)) continue;

    const sql = await readFile(join(MIGRATIONS_DIR, filename), 'utf-8');
    db.exec(sql);
    db.run('INSERT INTO _migrations (filename) VALUES (?)', [filename]);
    console.log(`applied: ${filename}`);
  }
}

// CLI entry — only runs when executed directly
if (import.meta.main) {
  const dbPath = process.env.DATABASE_URL ?? './dev.db';
  const db = new Database(dbPath, { create: true });
  await applyMigrations(db);
  console.log('migrations complete');
  db.close();
}

# IRS Circular 230 — Superhuman-cpa-OS Enforcement Rules

Circular 230 governs when a credentialed tax professional (CPA, EA, or attorney with PTIN)
must be involved in tax preparation. Superhuman-cpa-OS enforces this at two layers.

## Layer 1 — Claude System Prompt

Claude's system prompt contains the following mandatory rules. Claude MUST call
`flag_for_pro_review` when any of these conditions are detected:

- **Gray-area deductions:** Any deduction where IRS disallowance risk is non-trivial
  (e.g., home office with mixed-use signals, meals and entertainment, casualty losses)
- **Amended returns:** Any return that corrects a prior-year filed return
- **IRS correspondence:** Taxpayer mentions receiving a notice, audit, or inquiry
- **Non-resident situations:** Substantial presence test, dual-status returns, FBAR triggers
- **Complex pass-through income:** K-1s with unusual items, UBTI, passive activity losses
- **Tax treaty positions:** Any claim of treaty benefit
- **Estimated penalty exposure:** Where underpayment penalties appear likely
- **Circular 230 §10.34:** Any position that lacks a reasonable basis

When in doubt, call `flag_for_pro_review`. A pro can always approve and resume —
a missed gate cannot be undone after signing.

## Layer 2 — API Enforcement

`api` enforces the following status transition rules via the `X-Caller` header:

| Status transition                | Permitted caller   |
| -------------------------------- | ------------------ |
| `intake` → `ai_prep`             | `ai-worker`        |
| `ai_prep` → `pending_pro_review` | `ai-worker`        |
| `pending_pro_review` → `review`  | `practitioner`     |
| `review` → `signed`              | `practitioner`     |
| `signed` → `filed`               | `api` (staff only) |

Any caller attempting a transition outside this matrix receives `403 Forbidden`.

## Pro Eligibility

A pro may only review and sign returns when:

- `users.status = 'active'` (PTIN verified by staff)
- `users.role = 'pro'`

A pro with `status = 'pending_ptin_verification'` cannot access the task queue.
```
